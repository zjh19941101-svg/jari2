# Jari 2.0 Prototype

这是 Jari 2.0 的前端重构原型，不是在旧 `index.html` 上打补丁。

## 核心改动

1. 学生端首页改为「我的课程」。
2. 账号状态与课程状态分离。
3. 当前课程统一使用 `activeCourseId`。
4. QR/课程码只负责一次性加入课程。
5. 课程详情页内包含：签到、座位、公告、提问、记录。
6. TA 端按工作流组织：今日课堂、出勤、座位、公告、提问、补签、导出。
7. 公告与提问保留，但定位为「课堂事务沟通」，不是替代微信群的大群聊天。
8. 补签、导出、动态验证码、座位管理全部保留。

## 注意

这是前端架构原型，仍使用 `localStorage` 保存数据。  
正式版本建议把以下内容迁移到后端或 Firebase Cloud Functions：

- 动态验证码生成与验证
- 签到写入
- TA 权限认证
- 补签审批日志
- 异常签到检测
